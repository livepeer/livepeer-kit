# Getting Started

<Callout>
  These docs are written for a React developer building on livepeer.js. For
  Node.js or alternative frameworks, you can use the vanilla JS library.
</Callout>

## Installation

Install `@livepeer/react` and its `wagmi` and `ethers` peer dependencies.

<Tabs items={['npm', 'pnpm', 'yarn']}>
  <Tab>
```bash
npm i @livepeer/react wagmi ethers
```
  </Tab>
  <Tab>
```bash
pnpm i @livepeer/react wagmi ethers
```
  </Tab>
  <Tab>
```bash
yarn add @livepeer/react wagmi ethers
```
  </Tab>
</Tabs>


## Quick Start

### 1. Create a livepeer client

First, create a livepeer `Client` instance using [`createReactClient`](/react/client), and pass a provider to it.

```tsx
import { createReactClient, studioProvider } from '@livepeer/react';

const client = createReactClient({
  provider: studioProvider({ apiKey: 'yourStudioApiKey' }),
});
```

Note: If you choose to use Studio as a provider, you will need to configure an API key for the
`studioProvider` which is [CORS-protected API key](/react/providers/studio#apikey).

[Read more about client configuration](/react/client)

### 2. Wrap app with `LivepeerConfig`

Next, wrap the app with the [`LivepeerConfig`](/react/LivepeerConfig) component, passing the `client` to it. This
is typically included in `_app.js` for Next.js or `App.js` with Create React App, so that the `LivepeerConfig` React Context
is available across every component.

```tsx
import {
  LivepeerConfig,
  createReactClient,
  studioProvider,
} from '@livepeer/react';

const client = createReactClient({
  provider: studioProvider({ apiKey: 'yourStudioApiKey' }),
});

function App() {
  return (
    <LivepeerConfig client={client}>
      <SomeComponents />
    </LivepeerConfig>
  );
}
```

### 3. Enjoy!

Use hooks! Every component inside the `LivepeerConfig` is now set up to use the livepeer hooks.

```tsx
import { useLivepeerProvider } from '@livepeer/react';

function Livepeer() {
  const livepeerProvider = useLivepeerProvider();

  return <div>Using {livepeerProvider.getConfig().name}</div>;
}
```

Want to learn more? Continue reading the documentation for more details.
